WeldFind v5.0 — Manuale Tecnico SCL v5.0 · Python v5.0

WeldFind e' un sistema di rilevamento in tempo reale della posizione di saldatura (o buco) su cilindri rotanti o parti in movimento lineare. E' composto da due componenti principali:

ComponenteTecnologiaFunzione
Fb954_WeldFindV50SCL – S7-1500Algoritmo real-time: acquisisce campioni laser+encoder, rileva la saldatura durante il movimento
weld_viewer_v50Python / TkinterAnalisi offline, simulazione, grid search parametri, archivio SQLite, monitor real-time PLC
📌 Principio di funzionamento Il sensore laser misura la distanza dalla superficie del pezzo mentre ruota. La saldatura (o buco) produce un'anomalia nella distanza. L'algoritmo rileva questa anomalia confrontando ogni campione con una baseline locale adattiva calcolata su una finestra angolare scorrevole.

Condizione reset (SCL)

Parametri filtro

Parametro SCLDefaultDescrizione

I_MinLaserDelta0.1 mmVariazione laser minima. Per superfici lisce: ridurre a 0.001.

Baseline look-back angolare

FOR loop SCL – blocco D
FOR j := 0 TO iSamplesAcquired - 2 DO rAngDiff := ABS(arAngles[iCurIdx] - arAngles[j]) IF rAngDiff > 180.0 THEN rAngDiff := 360.0 - rAngDiff END_IF IF rAngDiff >= 1.0 AND rAngDiff <= I_BaselineWindowDeg THEN rWinSum += arSamples[j] rWinSumSq += arSamples[j] * arSamples[j] iWinN += 1 END_IF END_FOR
Calcolo mean e sigma (float32 REAL PLC)
rM := rWinSum / INT_TO_REAL(iWinN) rV := rWinSumSq / INT_TO_REAL(iWinN) - rM * rM IF rV < 0 THEN rV := 0 END_IF rS := SQRT(rV)
⚠ Finestra minima Se iWinN < 3, il PLC propaga i valori del campione precedente: rM := arMean[iCurIdx-1]. Accade tipicamente nei primissimi campioni dove non ci sono ancora abbastanza punti nella finestra.

Calcolo soglie adattive

Soglie POSITIVE (laser > baseline)
ThHi = rM + I_SigmaFactor × rS + I_MinAbsDeviation → soglia apertura cluster ThLo = rM + (I_SigmaFactor - I_HysteresisSigmas) × rS + I_MinAbsDeviation × 0.5 → soglia chiusura
Soglie NEGATIVE (laser < baseline)
ThHiNeg = rM - I_SigmaFactor × rS - I_MinAbsDeviation → soglia apertura ThLoNeg = rM - (I_SigmaFactor - I_HysteresisSigmas) × rS - I_MinAbsDeviation × 0.5

Parametri soglie

Parametro SCLDefaultEffetto
I_SigmaFactor3.0Moltiplicatore sigma. Range tipico: 1.5 (sensibile) – 4.0 (conservativo).
I_MinAbsDeviation1.5 mmScarto minimo assoluto. Circa 1/3 dell'altezza attesa della saldatura.
I_HysteresisSigmas0.5Abbassa la soglia di chiusura cluster. Evita chiusure premature su segnali con leggera ondulazione.

Logica cluster online

Blocco E – apertura
IF NOT bInCluster THEN IF (NOT I_FlatWaitEnable OR bFlatSurfaceFound) THEN IF bCheckPositive AND laser >= ThHi THEN bInCluster := TRUE; polarity := 0; count := 1 ELSIF bCheckNegative AND laser <= ThHiNeg THEN bInCluster := TRUE; polarity := 1; count := 1 END_IF END_IF

Parametri cluster

Parametro SCLDefaultEffetto
I_MinConsecutive3Campioni minimi per cluster valido. Aumentare per ridurre falsi positivi su picchi isolati.

bAngleExceeded diventa TRUE quando rTotalTravel > rDetStartDeg. Con I_DetectionStartDeg = 0.0, rDetStartDeg = I_BaselineWindowDeg (comportamento identico a v4.9).

Parametro v5.0: I_DetectionStartDeg

ValoreComportamento
0.0 (default)Dead zone = I_BaselineWindowDeg (retrocompatibile con v4.9)
15.0Detection abilitata solo dopo 15° di rotazione, indipendentemente dalla window baseline
> 0.0 qualsiasiSeparazione esplicita tra "quanti gradi aspettare prima di detection" e "dimensione finestra baseline"
✓ Quando usarlo Superfici con eccentricita' meccanica che produce un drift sistematico nei primi gradi. Con window piccola (2°) e deriva >0.15mm/°, la sigma e' quasi zero e ThHi = mean + solo mabs. Impostando I_DetectionStartDeg = 15° si lascia alla baseline il tempo di stabilizzarsi prima di abilitare la detection.

Post-detection travel v4.5

Con I_StopOnWeld=TRUE, il FB non va in Done immediatamente al rilevamento. Continua ad acquisire campioni per rPostDetTravelDegC = 2.0° (costante interna) dopo la detection, poi va in Done. Durante questi 2° il cluster e OutPosizioneAsse continuano ad aggiornarsi.

Note sull'aritmetica float32

Il PLC S7-1500 usa esclusivamente REAL (IEEE 754 single precision, 32 bit). La simulazione Python replica questo comportamento con use_float32=True.

🚬 Perche' float64 Python da' risultati diversi Con valori laser ~400mm, la somma di 20-30 valori float32 accumulati sequenzialmente differisce dalla somma numpy float64 a causa degli errori di arrotondamento. Questa differenza puo' spostare la soglia di pochi centesimi di mm – sufficiente per perdere o acquisire 1 campione al bordo del cluster.
✓ Soluzione implementata Il worker del grid search usa direttamente le soglie arThreshHigh/Low salvate dal PLC quando disponibili. Elimina completamente il problema di ricalcolo float32 e garantisce che il worker veda esattamente le stesse soglie del PLC reale.

Falsi positivi su superfici eccentriche v5.0

Causa radice (DB28160 anello eccentrico)

Il cilindro ha un drift meccanico di 7.7mm su 111° (eccentricita' anello). Con window=2° e sigma quasi zero (float32 satura con pochi campioni), ThHi = mean + solo 0.5mm. Il drift di 0.22mm/° supera la soglia dopo pochi gradi dalla dead zone.

Analisi: ThHi al momento di bAngleExceeded (window=2.8°, drift=0.22mm/°)
mean ≈ s0 + drift × W/2 = s0 + 0.22 × 1.4 = s0 + 0.31mm sigma ≈ drift × W / (2√3) = 0.22 × 2.8 / 3.46 = 0.18mm ThHi = mean + 2.8 × 0.18 + 0.5 = mean + 1.0mm laser[W] = s0 + drift × W = s0 + 0.62mm < ThHi → OK

Sui casi reali il problema si manifesta con drift piu' rapido o sigma ancora piu' bassa (valori identici float32 nella finestra). La soluzione e' aumentare la dead zone senza modificare la finestra baseline.

Soluzione: I_DetectionStartDeg = 15°

Nei test su 1299 acquisizioni DB28160: 60% delle early detection (<30°) corrette senza perdere saldature reali. Su DB28010 e DB28300 impatto trascurabile (<2% miss aggiuntivi).
